# Java 9–11 (Modules, var, HTTP Client)

> این نسخه‌ها زیرساخت Java مدرن را تثبیت کردند: ماژول‌ها، استنتاج نوع، و HTTP Client داخلی. Java 11 اولین LTS بعد از 8 است.

---

## 📖 مفاهیم

### Module System (Project Jigsaw — Java 9)

**توضیح:**

سیستم ماژول، JDK و برنامه‌ها را به واحدهای محصور با مرز صریح تقسیم می‌کند. هر ماژول یک `module-info.java` دارد که اعلام می‌کند چه چیزی نیاز دارد (`requires`)، چه پکیج‌هایی را در معرض دیگران می‌گذارد (`exports`)، و چه سرویس‌هایی ارائه/مصرف می‌کند (`provides`/`uses`).

انگیزه: قبل از ماژول، classpath یک «JAR hell» بزرگ و تخت بود؛ هر کلاس public از همه‌جا قابل دسترس بود و هیچ encapsulation در سطح jar وجود نداشت. ماژول‌ها **strong encapsulation** می‌آورند: پکیجی که export نشده حتی با reflection (بدون `opens`) قابل دسترس نیست.

نکته‌ی واقعی: بسیاری از برنامه‌های Spring Boot هنوز روی classpath (نه module path) اجرا می‌شوند چون اکوسیستم به‌کندی مهاجرت کرده. اما خود JDK کاملاً ماژولار شده و این روی `jlink` و تصاویر کوچک تأثیر دارد.

**چرا مهم است:**

با `jlink` می‌توان runtime سفارشی و کوچک ساخت (فقط ماژول‌های لازم) که برای کانتینرها مفید است. همچنین strong encapsulation امنیت و نگهداری را بهبود می‌دهد.

**مثال کد:**

```java
// module-info.java
module com.example.payment {
    requires com.example.common;      // وابستگی صریح
    requires transitive java.sql;     // هر کس مرا requires کند java.sql را هم می‌گیرد
    exports com.example.payment.api;  // فقط این پکیج public است
    // پکیج‌های داخلی دیگر مخفی می‌مانند
    provides PaymentGateway with StripeGateway; // ثبت سرویس
}
```

**نکات کلیدی:**

- پکیجی که export نشود، حتی public بودنش بیرون دیده نمی‌شود.
- `requires transitive` وابستگی را به مصرف‌کننده هم منتقل می‌کند.
- برای reflection (مثل frameworkها) به `opens` نیاز دارید.

### Collection Factory Methods (Java 9)

**توضیح:**

`List.of()`, `Set.of()`, `Map.of()` راهی مختصر برای ساخت مجموعه‌های **immutable** می‌دهند. این‌ها با `Collections.unmodifiableList` فرق دارند: کاملاً immutable‌اند، null نمی‌پذیرند، و حافظه‌ی بهینه‌تری دارند.

**مثال کد:**

```java
List<String> roles = List.of("ADMIN", "USER");   // immutable
Map<String, Integer> limits = Map.of("free", 10, "pro", 1000);

// roles.add("X"); // ❌ UnsupportedOperationException
// List.of("a", null); // ❌ NullPointerException
```

**نکات کلیدی:**

- نتیجه واقعاً immutable است؛ هر تغییری استثنا می‌دهد.
- null مجاز نیست (برخلاف ArrayList).
- برای ساخت سریع داده‌ی ثابت عالی است.

### Stream & Optional Additions (Java 9)

**توضیح:**

Java 9 چند متد مفید اضافه کرد:
- `Stream.takeWhile(pred)` — تا وقتی شرط برقرار است عناصر را بردارد، سپس قطع کند.
- `Stream.dropWhile(pred)` — برعکس: تا وقتی شرط برقرار است حذف کند، بقیه را نگه دارد.
- `Stream.iterate(seed, hasNext, next)` — نسخه‌ی محدودشونده.
- `Optional.ifPresentOrElse`, `Optional.stream`, `Optional.or`.

**مثال کد:**

```java
List<Integer> nums = List.of(1, 2, 3, 10, 4, 5);
System.out.println(nums.stream().takeWhile(n -> n < 5).toList()); // [1, 2, 3]
System.out.println(nums.stream().dropWhile(n -> n < 5).toList()); // [10, 4, 5]

Optional<String> name = Optional.of("Ali");
name.ifPresentOrElse(
    n -> System.out.println("سلام " + n),
    () -> System.out.println("ناشناس"));
```

**نکات کلیدی:**

- `takeWhile/dropWhile` روی stream مرتب معنا دارد (برخلاف filter که همه را بررسی می‌کند).
- `Optional.stream()` برای تخت کردن لیست Optionalها عالی است.

### Local Variable Type Inference — var (Java 10)

**توضیح:**

`var` اجازه می‌دهد نوع متغیر محلی را کامپایلر استنتاج کند. این فقط syntactic sugar در زمان کامپایل است؛ Java همچنان static typed می‌ماند و `var` نوع را در زمان کامپایل قطعی می‌کند (نه dynamic typing).

محدودیت‌ها: فقط برای متغیر محلی با مقداردهی اولیه؛ نه برای field، پارامتر متد، یا return type. نباید با `null` یا lambda بدون target type استفاده شود.

**چرا مهم است:**

boilerplate نوع‌های طولانی generic را کم می‌کند و خوانایی را در موارد بدیهی بالا می‌برد. اما استفاده‌ی بی‌رویه خوانایی را کم می‌کند.

**مثال کد:**

```java
var users = new ArrayList<User>();          // واضح: ArrayList<User>
var entry = Map.entry("k", 1);              // نوع طولانی پنهان می‌شود

// ❌ خوانایی کم: نوع از سمت راست معلوم نیست
var result = service.process();
// var x = null; // ❌ کامپایل نمی‌شود
```

**نکات کلیدی:**

- `var` نوع را در compile-time قطعی می‌کند؛ dynamic typing نیست.
- وقتی نوع از سمت راست واضح است استفاده کنید؛ وگرنه نوع صریح خواناتر است.
- فقط برای متغیر محلی.

### HTTP Client (Java 11)

**توضیح:**

`java.net.http.HttpClient` جایگزین مدرن `HttpURLConnection` قدیمی و وابستگی‌های خارجی مثل Apache HttpClient است. از HTTP/2 و WebSocket پشتیبانی می‌کند، API روان (builder) دارد و هم synchronous (`send`) و هم asynchronous (`sendAsync` با `CompletableFuture`) است.

**چرا مهم است:**

برای فراخوانی سرویس‌های خارجی بدون افزودن وابستگی. در میکروسرویس‌ها رایج است (هرچند Spring `WebClient`/`RestClient` معمولاً ترجیح داده می‌شود).

**مثال کد:**

```java
HttpClient client = HttpClient.newBuilder()
    .version(HttpClient.Version.HTTP_2)
    .connectTimeout(Duration.ofSeconds(5))
    .build();

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/users/1"))
    .header("Accept", "application/json")
    .GET()
    .build();

// همزمان
HttpResponse<String> response = client.send(
    request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.statusCode());

// ناهمزمان
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
    .thenApply(HttpResponse::body)
    .thenAccept(System.out::println);
```

**نکات کلیدی:**

- `HttpClient` thread-safe است؛ یک نمونه را reuse کنید.
- از HTTP/2 و async به‌صورت داخلی پشتیبانی می‌کند.
- timeout را همیشه تنظیم کنید تا از hang جلوگیری شود.

### String & Files Methods (Java 11)

**توضیح:**

متدهای مفید String: `isBlank()` (خالی یا فقط whitespace)، `strip()`/`stripLeading()`/`stripTrailing()` (Unicode-aware برخلاف `trim`)، `lines()` (Stream خطوط)، `repeat(n)`. همچنین `Files.readString()`/`Files.writeString()` برای کار ساده با فایل.

**مثال کد:**

```java
String text = "  سلام دنیا  ";
System.out.println(text.strip());      // "سلام دنیا" (Unicode-aware)
System.out.println("ab".repeat(3));    // "ababab"
System.out.println("  ".isBlank());    // true

Path p = Path.of("notes.txt");
Files.writeString(p, "خط اول\nخط دوم");
Files.readString(p).lines().forEach(System.out::println);
```

**نکات کلیدی:**

- `strip` به‌جای `trim` برای متن Unicode (مثل فارسی) بهتر است.
- `isBlank` با `isEmpty` فرق دارد (whitespace را هم در نظر می‌گیرد).

## 🎯 سوالات مصاحبه

### سوال ۱: تفاوت `var` با dynamic typing چیست؟

**سطح:** Mid / Senior
**تکرار:** زیاد

**جواب کامل:**

`var` فقط استنتاج نوع در زمان کامپایل است؛ کامپایلر نوع را از مقداردهی اولیه قطعی می‌کند و آن نوع برای همیشه ثابت می‌ماند. این کاملاً متفاوت با dynamic typing (مثل JavaScript `var`) است که در آن نوع در runtime می‌تواند عوض شود. در Java بعد از `var x = "hi";` نمی‌توانید `x = 5;` بنویسید چون x همان String است. Java static typed باقی می‌ماند و هیچ سربار runtime وجود ندارد.

**کد توضیحی:**

```java
var name = "Ali"; // نوع: String، قطعی در compile-time
// name = 42;     // ❌ خطای کامپایل
```

**نکته مصاحبه:**

Junior ممکن است فکر کند `var` مثل JS است. Senior تأکید می‌کند static typing حفظ می‌شود. Follow-up: «کجا `var` خوانایی را کم می‌کند؟»

---

### سوال ۲: Java Module System چه مشکلی را حل می‌کند و چرا کمتر استفاده می‌شود؟

**سطح:** Senior / Lead
**تکرار:** متوسط

**جواب کامل:**

ماژول‌ها سه مشکل را حل می‌کنند: (۱) strong encapsulation در سطح jar — قبلش هر کلاس public از همه‌جا دیده می‌شد؛ (۲) وابستگی صریح و قابل‌بررسی در زمان کامپایل و راه‌اندازی به‌جای classpath تخت؛ (۳) امکان runtime سفارشی کوچک با `jlink`.

اما در عمل اکوسیستم به‌کندی مهاجرت کرد چون مهاجرت کتابخانه‌های قدیمی پرهزینه بود، reflection نیاز به `opens` دارد و بسیاری از ابزارها فرض classpath می‌کردند. به همین دلیل اکثر برنامه‌های کاربردی (به‌خصوص Spring Boot) هنوز روی classpath اجرا می‌شوند، در حالی که خود JDK کاملاً ماژولار است.

**نکته مصاحبه:**

Lead به trade-off عملی و واقعیت مهاجرت اکوسیستم اشاره می‌کند، نه فقط تعریف کتابی. Follow-up: «`requires transitive` چه می‌کند؟»

### سوال ۳: تفاوت `List.of()` با `Collections.unmodifiableList()` چیست؟

**سطح:** Mid / Senior
**تکرار:** متوسط

**جواب کامل:**

`List.of()` یک لیست واقعاً immutable و مستقل می‌سازد که null نمی‌پذیرد و حافظه‌ی بهینه دارد. `Collections.unmodifiableList(original)` فقط یک **view غیرقابل‌تغییر** روی لیست اصلی می‌سازد؛ اگر کسی لیست اصلی را تغییر دهد، view هم تغییر می‌کند (نشت تغییرپذیری). بنابراین `List.of` برای ثابت‌های واقعی امن‌تر است.

**کد توضیحی:**

```java
List<String> base = new ArrayList<>(List.of("a"));
List<String> view = Collections.unmodifiableList(base);
base.add("b");
System.out.println(view); // [a, b] ← تغییر نشت کرد!

List<String> immutable = List.of("a"); // مستقل و امن
```

**نکته مصاحبه:**

Follow-up: «چطور یک کپی دفاعی immutable بسازی؟» (پاسخ: `List.copyOf(original)`).

### سوال ۴: HTTP Client داخلی Java 11 چه مزیتی نسبت به قدیمی دارد؟

**سطح:** Senior
**تکرار:** متوسط

**جواب کامل:**

`HttpURLConnection` قدیمی API دست‌وپاگیر، فقط HTTP/1.1، و فاقد async واقعی بود. `HttpClient` جدید: builder روان، HTTP/2 و WebSocket داخلی، پشتیبانی async با `CompletableFuture`، مدیریت بدنه با `BodyHandlers`، thread-safe و قابل‌reuse. بدون افزودن وابستگی خارجی، نیاز Apache HttpClient را برای موارد ساده برطرف می‌کند.

**نکته مصاحبه:**

Senior تأکید می‌کند که در اکوسیستم Spring معمولاً `WebClient`/`RestClient` ترجیح داده می‌شود اما برای کد بدون Spring این انتخاب خوبی است.

### سوال ۵: `takeWhile` با `filter` چه فرقی دارد؟

**سطح:** Mid / Senior
**تکرار:** متوسط

**جواب کامل:**

`filter` کل stream را بررسی می‌کند و هر عنصری که شرط را برآورده کند نگه می‌دارد. `takeWhile` به‌محض اولین عنصری که شرط را برآورده نکند **متوقف** می‌شود و بقیه را نادیده می‌گیرد (short-circuit). بنابراین روی stream مرتب رفتار متفاوت و کارآمدتری دارد.

**کد توضیحی:**

```java
List<Integer> nums = List.of(2, 4, 6, 7, 8);
nums.stream().filter(n -> n % 2 == 0).toList();    // [2, 4, 6, 8]
nums.stream().takeWhile(n -> n % 2 == 0).toList(); // [2, 4, 6] ← در 7 قطع
```

**نکته مصاحبه:**

Follow-up: «روی stream نامرتب چه می‌شود؟» (نتیجه غیرقطعی برای parallel).

## ⚠️ اشتباهات رایج

### اشتباه ۱: تغییر مجموعه‌ی ساخته‌شده با `List.of`

```java
// ❌ UnsupportedOperationException
List<String> list = List.of("a", "b");
list.add("c");
```

```java
// ✅ اگر تغییرپذیری لازم است
List<String> list = new ArrayList<>(List.of("a", "b"));
list.add("c");
```

**توضیح:** `List.of` immutable است. برای تغییر، آن را در ArrayList بپیچید.

---

### اشتباه ۲: استفاده‌ی بیش از حد `var`

```java
// ❌ نوع نامشخص، خوانایی کم
var x = service.fetch();
var y = process(x);
```

```java
// ✅ نوع صریح وقتی واضح نیست
PaymentResult x = service.fetch();
```

**توضیح:** `var` فقط وقتی نوع از سمت راست واضح است خوانایی را بهتر می‌کند.

---

### اشتباه ۳: ساخت HttpClient جدید برای هر request

```java
// ❌ هزینه‌ی ساخت connection pool در هر فراخوانی
for (var url : urls) {
    HttpClient.newHttpClient().send(...);
}
```

```java
// ✅ یک نمونه‌ی reusable
HttpClient client = HttpClient.newHttpClient();
for (var url : urls) client.send(...);
```

**توضیح:** `HttpClient` thread-safe و سنگین است؛ یک‌بار بسازید و reuse کنید.

---

### اشتباه ۴: استفاده از `trim` برای متن Unicode

```java
// ❌ trim فقط کاراکترهای <= U+0020 را حذف می‌کند
String s = "\u2009متن\u2009".trim(); // فضاهای Unicode باقی می‌مانند
```

```java
// ✅
String s = "\u2009متن\u2009".strip();
```

**توضیح:** `strip` بر اساس `Character.isWhitespace` کار می‌کند و Unicode-aware است.

## 🔗 ارتباط با سایر مفاهیم

- Module System با **jlink** و ساخت تصاویر کوچک برای **Docker/K8s** ترکیب می‌شود.
- HTTP Client با **microservices** و فراخوانی سرویس‌به‌سرویس مرتبط است و در Spring با **WebClient/RestClient** جایگزین می‌شود.
- `var` با **records** و **lambda** برای کد مختصرتر در نسخه‌های بعدی ترکیب می‌شود.
- متدهای Stream جدید مکمل **Stream API** فصل Java 8 هستند.
